# Judging Strategy

## Primary Target

Built with Python Hackathon

- Low participant count.
- Five non-cash award slots.
- Strong alignment with Python functionality, creativity, usability, and explanation.
- Submit by June 20, 2026 at 5:00 PM PDT.

## Secondary Target

Mind the Product: Everyone Ships Now

- Higher competition, but has cash prizes.
- CampusPulse has a concrete user, measurable value, and a shipped dashboard.
- Requires a deployed URL, a 2-3 minute demo video, and Novus.ai installation.

## Later Target

VoltHacks

- Low participant count and small cash prizes.
- Stronger if we add a lightweight IoT angle or mock sensor device flow.
- Deadline is September 5, 2026.

## Positioning

CampusPulse should be described as a student-built sustainability product, not just a dashboard. The strongest narrative is:

> A UCI student noticed that campus buildings have changing occupancy patterns, but energy usage is hard to inspect. CampusPulse makes those invisible patterns visible and turns them into concrete facilities actions.

## What Judges Should Notice

- The app works immediately.
- Python does meaningful computation.
- The UI is polished and product-like.
- The project has a specific user and measurable impact.
- The README and demo explain the logic clearly.
